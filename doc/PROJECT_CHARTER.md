## 2. 一句话定义

**OpsPulse** 是面向 Java 后端团队的 **Issue-to-Deploy 虚拟 FDE 开源脚手架** — 通过 MCP 将 GitHub Issue 解析为可执行交付契约，驱动 AI 开发 + CI/CD 自动验证上线，并以 FDE 土路铺高速机制将企业定制沉淀为可复用资产。

---

## 3. 使命与愿景

| 层级 | 内容 |
|------|------|
| **使命** | 让请不起 Palantir FDE 的中小 Java 团队，也能拥有 Issue 驱动的工程交付流水线 |
| **愿景（24 月）** | 成为国内「后端 Issue-to-Deploy」细分领域的开源标杆作者 |
| **长期愿景** | 以开源工具 + 垂直 IP + 可持续技术服务形成独立开发者全职收入 |

---

## 4. 赛道定位（绝不摇摆）

### 做

- Java 后端 / 微服务 / DevOps 工程 AI 提效
- Issue-to-Deploy 全链路脚手架
- MCP + GitHub Issue + CI/CD（Harness/GHA）集成
- 工程流水线、批量自动化、中间件治理
- 垂直实战内容（公众号 + GitHub）

### 不做

- 通用 AI 编辑器测评（Cursor/Trae 追更）
- 前端 AI、通用 Vibe Coding 热点
- 短期流量爆款、无产品沉淀的水文
- 纯咨询外包（无产品化回流）
- 需要重算力/重生产环境才能验证的功能

---

## 5. 目标用户

### 核心用户画像

- **角色**：3–10 人后端团队的 TL、平台工程师、On-call 负责人
- **团队规模**：50–500 人公司
- **技术栈**：**JDK 1.8 微服务** + GitHub + CI/CD；企业统一提供 **JDK8 运行时基础镜像**（OS+JVM+公共依赖，不含业务应用）
- **痛点**：需求在 Issue、代码在 PR、上线在流水线 — 三段割裂；AI 工具只能补全单文件，无法驱动全链路

### 非目标用户

- 无 CI/CD 基建的初创团队（试用门槛过高）
- 纯前端团队
- 已自建完整 Internal Developer Platform 的大厂
- 愿付 Palantir 级 FDE 费用的巨头企业

---

## 6. 最终资产形态（24 个月后）

1. **1 个垂直开源工具**：`opspulse`（MCP Server + Issue 契约 + 镜像化流水线模板）
2. **1 套行业专属资产**：Issue Spec、Harness 模板库、Agent Prompt 库、实战文章库
3. **垂直开发者私域 IP**：公众号 + GitHub 社区 + 技术社群
4. **可长期接单的技术服务能力**：企业流水线定制（强制 48h 产品化回流）

---

## 7. 商业模式

```
一线真实痛点（用户访谈 + Issue 反馈）
  → 自研开源工具 Day1 交付
  → 公众号内容沉淀 & 引流
  → GitHub Star & 私域沉淀
  → 分层商业化（资料包 / 社群 / Pro 插件 / 咨询）
  → 需求回流再迭代（产品化率 ≥ 0.5）
```

---

## 8. 核心 KPI（决策优先级）

1. **产品化率** ≥ 0.5（L1/L2 抽象数 / L3 交付数）
2. **TTFV** < 15 分钟（用户从 clone 到跑通首个命令）
3. **Day1 交付率** ≥ 80%（每周有可运行交付的占比）
4. GitHub Star / 收入（次要，作健康度参考）

---

## 9. 项目约束

| 约束 | 说明 |
|------|------|
| 单人维护 | 24 个月内不假设全职协作者 |
| 本地可验证 | 所有演示必须本机 Docker 跑通 |
| 不追热点 | 不做 AI 版本测评 |
| 产品化纪律 | L3 定制 48h 内评估抽象；产品化率 < 0.2 停接定制 |
| 免费版完整 | 免费版不得人为残废 |
